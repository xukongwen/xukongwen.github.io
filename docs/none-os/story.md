# 缘起

None-OS的起源说起来略有点复杂。

只能一点一点说。

![](https://s3.ax1x.com/2021/01/31/yA6tT1.jpg)

None-OS可以说是一个小说的产物，也可以说是一个游戏的产物。或者毋宁说是，为了丰富游戏世界观而写作的小说的附带品。

我很喜欢“附带品”这个词儿，我记得儿时看“阿拉蕾”的时候经常里面买点东西，都会附送一点什么附带品。

当下这个小说甚至和游戏关系甚小，为了不给自己和读者一种唐突之感，我并没有着手写和游戏完全相关的小说，而是写一个给游戏世界观起到一点润滑作用的铺垫。

那么在这个铺垫里面，有那么一个写作者，因为常年找不到合适的写作工具而迟迟无法下笔。整个故事就是这个无法下笔，但又天天灵感四跃的写作者的日常。

于是他决定在正式写作之前，他要严肃的解决写作工具这件事。

后来他为了这个“严肃”的议题，大约花了十年时间，也终未解决，到结束的时候，甚至他的小说还未动笔。

那么我正好是写在这里的时候忽然找到了我落灰多年的树莓派，也许我可以体验以下制作一个自己的写作工具是个什么感觉。

我的树莓派一直放在“灵魂石”上面，本来买她也是充当灵魂石的“灵魂”，也就是存储和显示。灵魂石这东西是我游戏里面的一个设计，说起来异常简单，你可以理解为，这东西就是个移动硬盘，但装的内容是“你的一切”，你的记忆，你的性格，你的思维，你的灵魂。

顺带一提，这个游戏项目彻底被我做死了，成为了[一段记忆]()，封印在灵魂石里，等待着未来发生什么不测，也许会被再次激活。

你看上面这两个家伙，一位未来末世的女士正要给十字架上的男士复活，这多少有点像是基督被钉死之后，某个虔诚的信徒，希望他能再次醒过来。

灵魂石，只要通了电，哪怕是充电宝，你可以可以醒过来，如果你这东西还能运作的话。

![](https://s3.ax1x.com/2021/02/04/y1xUlF.jpg)

当时我们设计了很多款式，但看起来都不怎么结实。

![](https://s3.ax1x.com/2021/02/04/y1xokt.png)

这是我同事帮我制作的灵魂石真身，那时我忽然发现有树莓派这么个东西，大小正好放在灵魂石上头，而且都是绿色的，简直完美无缺。

![](https://s3.ax1x.com/2021/02/04/y1xjmj.png)

记得，当初兴奋地把树莓派放上去，结果一下子就阵亡了。因为那时我是个蠢货，不知道要尽量绝缘，把电路板放在了一个超级不绝缘体上，瞬间树莓派啪的一声就死了。

所以后来我在上面包了一块塑料，又再买了一块派。好在这东西不那么贵。

之后我人生的第一次学习程序就在这上面，学了一辆个月的Python，终于在我脑壳里打开了一个小洞，哦，原来这就是所谓编程。好像没有想像的那么困难。当时甚至感觉电脑真的是蠢爆了。

我做的第一个小项目就是用python的数据分析功能，分析《伤寒杂病论》，因为那会子我正因为拿捏不到张仲景到底如何下药感到万分苦恼。于是想借助数据分析，看看里面的门道。

多亏我网路上认识了一个好心帮我的正儿八经的python程序员，得以作出了这个小项目。

Github

分析了一路之后，我彻底爱上了他老人家，现在依旧认为他是人类已知最伟大的医生和医学理论家。

而我的第二次程序尝试就是试图在ios上制作一款方便查阅伤寒论的程序。

Github

去年（2020）我莫名其妙买了一个自己组装的机械键盘，手感出奇的美妙，乃至于我每天不敲一会字，我就浑身不舒服。以前我一直用一款双段落的机械键盘，声音大的不得了，但是这个新家伙只有一个段落，而且声音变得非常轻巧和清脆，于是整个书写过程变成了行云流水，无法停止。

那么，话说回来，因为这灵魂石，这游戏，这小说，这机械键盘，这程序，这张仲景，我开始试图将他们组合成一体，一个恍恍惚惚间，我期待的那么一个“终极写作工具”。



![](https://s3.ax1x.com/2021/02/02/yukGLj.jpg)

**最初的版本还是运行着Emacs**

我开始着手在树莓派上搭建这么一个写作工具。

顺着这个思路，我很快发现了两个“上古写作神器”，当然他们并不是为了我这种写小说的人准备的，而是给程序员们敲代码准备的。

一个是Emacs，一个是Vim。

我先是下载了Vim，因为那会儿正在家里玩赛博朋克2077这个奇葩（实则伤了我的心），里面的主角叫做“V”，那么好吧，就先试试V君吧。

第一次接触下来，想死的心都有了。

为什么世界上会有这种逆天的反人类的操作！

之前我的写作工具大多是Word，各种Markdown（最后停留在ulyseses），比如国内的纯纯写作之类的，甚至网页上的Notion之流，还有专门写剧本的celx之流，几乎我把大部分市面上能见过的文字编辑器都用过一遍。

猛然间用了V君，真的让我百思不得其解，为啥你非要这样操作呢？

Vim不支持鼠标，而且分好多个模式，上下左右这种最平常的操作竟然改成了hjkl，然后什么向前跳几个字符，向后跳几个字符等等，一大堆眼花缭乱的命令，和命令的组合简直对我造成了一万电暴击伤害。

我不愿意！

我平静了几天心情之后，开始不死心，就战战兢兢的开始尝试Emacs，因为有了心里准备，知道这家伙可能是一路人，不会给我好果子吃。

确实，但是貌似稍微好一点。

首先，打开这家伙，我看了第一眼就想关机，真的，我不开玩笑。

你好歹给我一个顺服的写作环境吧，这算啥呀。

我硬着头皮发现，哦，这玩意不装插件是别想用了，于是摸索着装了一些插件。

恩，一个全屏的，一切剧中的效果很快做出来了，很不错。

而且Emacs没有Vim那些该死的模式，就一个写作模式，这，那么就他了！

后来慢慢了解说Emacs不是文字编辑器云云，是什么操作系统云云，我也没太大感觉，知道插件非常丰富，但我可不觉得他和操作系统有任何可比性，就是一个运行lisp的程序环境吧。

于是我开始打造一个能在Pi Zero上跑起来的写字系统，也就是进入官方提供的Pi Lite（纯文字精简版，非GUI）系统，然后一通安装中文显示和中文输入等等必须的中文写作条件，并且去掉了众多开机信息，最后当然是开机直接进入Emacs，直接开写。

一通这样的操作下来，感觉上好像真有那么一回事了！

开机logo过后，直接就进入写作模式，这妙啊！而且还是完全自己定制的写作工具，这，美啊！

我当时爱Emacs要死要活的，认定这是世界上现存最伟大的写作程序，甚至是最伟大的程序之一。后来慢慢了解她的历史，她和linux，unix的渊源，更是爱的五体投地。

但是，惊醒我的，是曹雪芹老师。

在Emacs里面我试着打开自己的小说之类的东西，都挺顺畅，后来我打开了个我很早写的一个小说练习，大概是15万字，忽然觉得怎么略略有那么一点卡顿。

我为了实验一下更多的字数文件，就下载了一个《红楼梦》，大约不到80万字。

好了，光是打开就花了30秒左右，然后当我试图编辑的时候，已经卡到不能操作的地步了。

V君这时在我背后闪耀，我抱着期待用Vim打开红楼梦，2秒...并且打字丝毫没有卡顿...

sudo apt remove emacs...
